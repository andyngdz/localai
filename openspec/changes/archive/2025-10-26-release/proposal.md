# Change: Semantic Release Process

## Why

Manual version management and changelog generation is error-prone and time-consuming. Automated semantic versioning ensures consistent releases.

## What Changes

- Integrate semantic-release for automated versioning
- Configure Conventional Commits format parsing
- Auto-generate categorized changelogs
- Create GitHub releases with platform artifacts
- Support main (stable) and develop (beta) branches

## Impact

- Affected specs: release
- Affected code: CI/CD configuration, release scripts
